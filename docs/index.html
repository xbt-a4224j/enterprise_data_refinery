What it ships

The 70% every other repo skips.

Any weekend project can call a model and hand you JSON. Production needs the layer that decides whether to trust it.

Fail-closed gate

Typed eval checks run before publish. If the data can't pass, it quarantines instead of shipping. Bad extraction is caught, not served.

Drift detection

A source silently changes format? Schema drift is flagged and the mapping re-derived. Value drift is tracked over time.

Provenance

Every published row answers: which source, run, mapping version, and checks produced it. It can defend itself to an auditor.

Cost accounting

$0 local vs. would-be-Claude, per run. The mapping cache skips the model on unchanged inputs — spend tracks real change.

Pack system

A new document domain is a Pack — config, a schema, its checks. Zero core changes. Or add one from a sample in the UI, no code.

Observability

A real admin surface: run history, live logs, a gate-failure browser, a canonical explorer. Diagnosable, not a black box.

The pipeline

Fetch. Extract. Gate. Publish.

Every run, every Pack — with the eval gate as the hinge between "extracted" and "trusted".

01 / FETCH

Pull & cache

Fetch raw documents on a schedule. Unchanged inputs skip the model entirely via the mapping cache.

02 / EXTRACT

Model maps

A local or frontier model maps each document into the Pack's canonical schema, with a confidence signal.

03 / GATE

Trust or quarantine

Typed checks decide. Pass → published with provenance. Fail → quarantined + alerted, last-good untouched.

04 / SHIP

Publish & watch

Clean drops export to a data repo; drift is monitored across drops for the longitudinal view.
