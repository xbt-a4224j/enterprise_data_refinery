Enterprise Data Refinery — how it works

Hostile documents in, trustworthy structured data out. The model does the extraction; the trust-and-ops layer is the product.

01What it does

The LLM call is the easy 30%. The mass of the system — and the reason an enterprise would run it — is the trust-and-ops layer wrapped around the model.

02The pipeline — fail-closed

Every run, every Pack. The eval gate is the hinge: pass and it publishes with provenance; fail and it quarantines and alerts. Nothing bad ships.

03The Pack model — swappable domains

The core never changes. A document domain is a Pack: a source adapter, a target schema, its eval checks, and its output workflows. Ship three, write your own.

04Adding a Pack — two tiers

Engineers drop in a directory. Non-engineers define a Pack by example: upload a sample, the model proposes the schema and checks, you tweak and save. Both write the same files.

05Topology — $0 local, cloud-portable

One Docker Compose stack on your machine, no paid APIs. Twelve-factor, so the same containers lift to a managed cloud host when there's a reason to pay.
